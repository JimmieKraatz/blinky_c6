# blinky_c6

ESP-IDF app for ESP32-C6 that drives an LED using a waveform-based state machine. A single button controls run/pause and a wave-selection menu. The LED can display square, saw, triangle, or sine patterns, with configurable timing via `menuconfig`.

This repo is split into portable core logic and ESP-IDF-specific integration:
- `core_sm`: framework-agnostic FSM/HSM engines
- `core_blinky`: framework-agnostic blinky model/menu logic
- `blinky_idf`: ESP-IDF + hardware integration (button, LED PWM, app state machine)

**Decoupling Refactor**
- Goal: isolate reusable logic from ESP-IDF/HAL code so core behavior can be tested and reused independently.
- `components/blinky/fsm_engine.h` -> `components/core_sm/fsm_engine.h`
- `components/blinky/led_model.*` + `menu_logic.*` -> `components/core_blinky/`
- `components/blinky/button.*` + `led_sm.*` + Kconfig integration -> `components/blinky_idf/`
- Unit tests moved with ownership:
  `core_sm/test/*`, `core_blinky/test/*`, `blinky_idf/test/*`

**Blinky Behavior**
- LED brightness follows a waveform generated by the state machine.
- Button input drives state transitions.
- The app entry point initializes the LED state machine and steps it in a loop (`main/main.c`).

**How To Interact (Default Pins)**
- LED output: `GPIO_NUM_14` (PWM, active-low, 5 kHz).
- Button input: `GPIO_NUM_9` (active-low with pull-up).

**Controls**
- Short press:
  Pause or resume the waveform.
- Long press (~3 s):
  Enter/exit the wave selection menu.
- While in menu:
  Short press cycles wave type (square, saw up/down, triangle, sine). The LED
  continues animating using the currently selected wave so it can be observed
  without a serial monitor.

The app logs state transitions and wave names to the serial console.

**Configuration (menuconfig)**
Config options are under `Component config -> Blinky` (from `components/blinky_idf/Kconfig`):
- LED GPIO, Button GPIO, active-low setting
- Poll interval, debounce count, long-press duration
- Wave period (target cycle time)
- Sine LUT max steps and saw/triangle step size
- PWM frequency
- Startup waveform
- Boot pattern enable and pulse duration
- Intensity logging (enable/disable per-update LED percentage logs)

**Requirements**
- ESP-IDF installed and activated (e.g. `export.sh` on Linux/macOS)
- A configured target for ESP32-C6

**Configure Target**
```bash
idf.py set-target esp32c6
```
VS Code tasks use the `idf.target` setting in `.vscode/settings.json`.

**Build**
```bash
idf.py -D CCACHE_ENABLE=1 build
```

**Flash and Monitor**
```bash
idf.py -p /dev/ttyUSB0 flash monitor
```
Replace the port with your device (for example `/dev/ttyACM0` on Linux or `COMx` on Windows).

**Run Unit Tests (Unity via ESP-IDF Unit Test App)**
Build the Unit Test App and point it at this repo's components via `EXTRA_COMPONENT_DIRS`.

Option A (terminal):
```bash
idf.py -C $IDF_PATH/tools/unit-test-app -B $PWD/build/unit-test-app -D EXTRA_COMPONENT_DIRS=$PWD/components -D "SDKCONFIG_DEFAULTS=$IDF_PATH/tools/unit-test-app/sdkconfig.defaults;$PWD/test/unit-test-app.sdkconfig.defaults" set-target esp32c6
idf.py -C $IDF_PATH/tools/unit-test-app -B $PWD/build/unit-test-app -D EXTRA_COMPONENT_DIRS=$PWD/components -D SDKCONFIG=$PWD/build/unit-test-app/sdkconfig -D "SDKCONFIG_DEFAULTS=$IDF_PATH/tools/unit-test-app/sdkconfig.defaults;$PWD/test/unit-test-app.sdkconfig.defaults" -D CCACHE_ENABLE=1 -T core_sm -T core_blinky -T blinky_idf build
idf.py -C $IDF_PATH/tools/unit-test-app -B $PWD/build/unit-test-app -D SDKCONFIG=$PWD/build/unit-test-app/sdkconfig -D "SDKCONFIG_DEFAULTS=$IDF_PATH/tools/unit-test-app/sdkconfig.defaults;$PWD/test/unit-test-app.sdkconfig.defaults" -T core_sm -T core_blinky -T blinky_idf -p /dev/ttyACM0 flash monitor
```

At the Unity prompt, run all tests with `*` then Enter.

Option B (VS Code Tasks):
- `IDF: Set Target (tests)`
- `IDF: Build (tests)`
- `IDF: Flash+Monitor (tests)`
